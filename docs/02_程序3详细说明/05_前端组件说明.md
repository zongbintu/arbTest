### 3.2.1 折溢价阈值设置功能

**功能说明**：用户可自定义开仓/平仓阈值，设置持久化到 localStorage。

**开仓逻辑**：
- **折价越大越好**（负数绝对值越大），如 -0.8%、-1%、-4% → 低价买入，开仓信号
- 阈值默认：-0.5%（折价超过 0.5% 才开仓）

**平仓逻辑**：
- **折价小或溢价**，如 -0.1%、0%、+2% → 利润空间小，平仓信号
- 阈值默认：2.0%（溢价超过 2% 才平仓）

**过滤公式**：
```javascript
f.rt_premium < premiumThreshold.value || f.rt_premium > premiumUpperThreshold.value
```

**localStorage 持久化**：
```javascript
localStorage.setItem('premiumThreshold', '-0.5')       // 开仓阈值
localStorage.setItem('premiumUpperThreshold', '2.0')    // 平仓阈值
```

**UI 组件**：
- 阈值设置弹窗（`n-modal`）
- 折价触发标签（绿色，`type="success"`）
- 溢价触发标签（红色，`type="error"`）

### 3.3 Analysis（深度分析）

**文件**：`frontend/src/views/Analysis.vue`

**路由**：`/analysis`

**职责**：单基金的实时估值曲线 + 溢价率柱状图 + 测算沙盘

**核心功能**：
1. **雷达模式**：全场套利机会实时雷达（未选择基金时显示）
   - 显示所有触发开仓/平仓信号的基金
   - 可自定义折价/溢价阈值
   - 阈值设置持久化到 localStorage
2. **详情模式**：专业狙击工作站（选择基金后显示）
   - 主图：基金现价（蓝线） vs 实时估值（橙线）
   - 副图：溢价率红绿柱状图（0 轴为中心）
   - 日期选择器：回看最近 30 天
   - 测算沙盘：手动输入假设价格，实时推演溢价率
   - 五档盘口：实时显示买卖盘深度
   - 摘要栏头：顶部 `T-1估值ETFF收盘`，精准对齐显示估值日相对基准日的涨跌幅（无需显示权重，保留关键涨跌幅数据），`T-2`等历史数据排版取消冒号，上下完美对齐。
   - **核心执行面板 (`TransactionPanel`)**：
     - **估值算法切换**：包含“ETF实时估值”（默认必选）、“期货校准估值”、“纯期货估值”三个复选框。如果基金不包含期货对应标的（如 XOP），后两者自动变灰不可用。
     - **动态按键布局**：
       - 当只勾选“ETF实时估值”或全不勾选时：采用常规布局（显示 4 个下单按键）。
       - 当勾选了“期货校准”或“纯期货”任何一个时：面板自动横向扩展，采用左中右三列、上下对齐的 6 按键布局，分别对应不同算法的执行路径。
     - **双通道下单引擎**：明确展示 银河QMT / 通达信，引擎直接调用自 `arbcore` 统一底座中的 `TradeManager`（与程序1底层共享，确保完全一致性，无隔离风险），摒弃了旧版的 8888 端口下拉框，直连底层。

```javascript
function calculateRedeemDate(tradeDate) {
  const date = parseDate(tradeDate)
  let workdaysAdded = 0
  while (workdaysAdded < 3) {
    date.setDate(date.getDate() + 1)
    if (date.getDay() !== 0 && date.getDay() !== 6) {  // 跳过周末
      workdaysAdded++
    }
  }
  return formatDate(date)
}
```
